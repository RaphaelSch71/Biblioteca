# Contributing

Thank you for your interest in contributing to Biblioteca.

## Basic Workflow

1. Fork the repository
2. Create a feature branch from `main`
3. Make focused changes with clear commits
4. Run local checks/tests relevant to your changes
5. Open a Pull Request with context and screenshots (for UI changes)

## Branch Naming (Suggested)

- `feat/<short-description>`
- `fix/<short-description>`
- `docs/<short-description>`

## Pull Request Guidelines

- Keep PRs small and scoped
- Describe **what** changed and **why**
- Link related issues (e.g., `Closes #12`)
- Include before/after screenshots for UI updates
- Ensure no secrets or credentials are committed

## Reporting Issues

Please use the GitHub issue templates for:

- Bug reports
- Feature requests

## Code of Conduct

Be respectful and constructive in all discussions and reviews.
